# [백엔드] 매장 관리자 및 디자이너 인증 로직 동기화 요청

## 1. 현재 문제점 (Current Issues)
프론트엔드는 **[매장 로그인(사업자 번호+비밀번호) -> 디자이너 선택 -> 디자이너 PIN]**의 2중 구조로 개편되었으나, 현재 백엔드 API는 다음과 같은 이유로 연동이 불가능합니다.

*   **회원가입 로직 미비**: `admin_signup_page` 뷰에 POST 데이터 처리 로직이 구현되어 있지 않아, 사용자가 회원가입 폼을 제출해도 실제 DB 저장이 이루어지지 않습니다.
*   **로그인 검증 필드 불일치**: 현재 `partner_verify` API는 여전히 4자리 PIN 번호 하나만 받는 구조로 되어 있습니다. 프론트엔드에서 전송하는 사업자 번호 정보를 무시하고 있어 특정 매장을 필터링할 수 없습니다.
*   **인증 흐름 충돌**: 매장 관리자의 비밀번호(Password)와 디자이너의 PIN 번호 검증 로직이 분리되어 있지 않아, 대규모 사용자 발생 시 인증 오류가 발생할 위험이 큽니다.

## 2. 요청 사항 (Requirements)

### 2.1 매장 관리자 회원가입 처리 (POST)
*   **대상 모델**: `AdminAccount`
*   **필요 필드**: `biz_number` (사업자 번호), `store_name` (상호명), `password` (비밀번호 - `make_password` 필수)
*   **작업**: `admin_signup_page` 뷰에서 POST 요청을 처리하여 계정을 생성하도록 보완 부탁드립니다.

### 2.2 로그인 API (`partner_verify`) 고도화
*   **변경 사항**: API 호출 시 `biz_number`(매장 특정용)와 `designer_id`(디자이너 특정용)를 조건부로 받아야 합니다.
*   **검증 단계**:
    1.  **매장 로그인**: 사업자 번호로 `AdminAccount`를 찾고 비밀번호 검증.
    2.  **디자이너 로그인**: 선택된 `designer_id`에 해당하는 디자이너의 PIN 검증.

### 2.3 디자이너 리스트 API 구현 (GET)
*   **경로**: `/api/v1/designers/` (또는 적절한 경로)
*   **기능**: 현재 세션에 활성화된 `active_shop` (매장)에 소속된 디자이너 목록을 JSON으로 반환해야 합니다.
*   **반환 데이터**: `id`, `name`, `profile_image` 등.

## 3. 비즈니스 로직 참고
*   본 서비스는 **미용실(Shop)이 로그인된 상태**를 기본 전제로 합니다.
*   미용실 로그인 후 해당 매장의 디자이너가 자신의 PIN으로 한 번 더 인증하여 **디자이너별 전용 고객 대시보드**에 접근하는 것이 핵심입니다.
