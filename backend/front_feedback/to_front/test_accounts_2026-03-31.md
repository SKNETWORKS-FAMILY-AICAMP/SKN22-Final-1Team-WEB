## 1. 기준 환경

현재 이 문서는 **Supabase 원격 DB 기준**으로 재정리된 테스트 계정 안내입니다.

- 0331 기준으로 아래 테스트 계정은 Supabase DB에 시드 적용을 완료했습니다.
- 같은 계정은 로컬 DB에서도 `seed_test_accounts`로 다시 만들 수 있습니다.
- 따라서 지금은 `로컬 전용 테스트 계정`이 아니라, **로컬/Supabase 공용 검증 계정**으로 보는 편이 맞습니다.

로컬 DB를 새로 준비해야 할 때만 아래 명령을 실행하면 됩니다.

```powershell
C:\Users\Playdata\AppData\Local\miniconda3\envs\mirrai\python.exe manage.py seed_test_accounts
```

참고:

- Supabase 기준 수동 검증은 이미 시드된 상태이므로, 일반적인 브라우저 확인 단계에서는 다시 실행할 필요가 없습니다.
- 로컬 DB를 새로 만들었거나 데이터를 초기화한 경우에만 재실행하면 됩니다.
