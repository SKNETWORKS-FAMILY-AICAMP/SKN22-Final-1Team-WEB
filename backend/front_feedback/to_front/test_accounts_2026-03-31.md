## 1. 기준 환경

현재 이 문서는 **Supabase 원격 DB 기준**으로 정리한 테스트 계정 안내입니다.

- 아래 계정과 샘플 고객은 Supabase DB에 적재되어 있습니다.
- 동일한 상태는 로컬 DB에서도 `seed_test_accounts` 명령으로 다시 만들 수 있습니다.
- 따라서 본 문서는 `로컬 전용 테스트 계정`이 아니라 **로컬/Supabase 공용 검증 계정 안내**로 보시면 됩니다.

## 2. 계정 생성 명령

로컬 DB를 새로 준비했을 때만 아래 명령을 실행하면 됩니다.

```powershell
C:\Users\Playdata\AppData\Local\miniconda3\envs\mirrai\python.exe manage.py seed_test_accounts
```

참고:

- Supabase 기준 수동 검증은 이미 시드가 들어간 상태를 기준으로 진행하면 됩니다.
- 로컬 DB를 초기화했을 때만 다시 실행하면 됩니다.

## 5. 권장 확인 시나리오

- shop 로그인 후 디자이너 선택 화면이 보이는지 확인
- `매장 전체 대시보드로 이동`으로 owner 화면에 들어갈 수 있는지 확인
- 고객 목록에서 배정 고객과 미배정 고객이 모두 보이는지 확인
- `한서` 고객으로 신규 배정 테스트가 가능한지 확인
- `최하나`, `이도훈` 고객으로 재배정 테스트가 가능한지 확인
- `최하나`, `이도훈`, `윤아라` 고객 상세에서 설문/촬영/분석/추천 데이터가 보이는지 확인
- `트렌드 리포트`에서 스타일 분포가 비어 있지 않은지 확인

## 6. 비고

- 이 문서는 특정 로컬 DB 파일을 공유하는 문서가 아니라, 같은 테스트 상태를 재현하기 위한 시드 기준 문서입니다.
- Supabase 원격 DB에서 테스트할 경우 실제 데이터가 변경될 수 있으므로, 작업 전후 상태를 함께 확인하는 것이 안전합니다.
