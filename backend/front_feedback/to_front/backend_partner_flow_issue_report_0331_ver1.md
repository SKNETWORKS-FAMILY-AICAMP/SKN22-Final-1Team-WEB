# 파트너 센터 흐름 문제점 및 요청사항 리포트

작성일: 0331  
버전: ver1

## 1. 개요

현재 `partner` 화면과 `customer` 분석 흐름이 서로 섞여 보이는 구간이 있어,
프론트에서 확인한 화면 기준과 백엔드의 현재 연결 기준을 함께 점검할 필요가 있습니다.

이번 리포트는 화면 디자인 자체보다,
`로그인`, `분석 시작하기`, `사진 촬영`, `취향 설문`, `추천 결과`가 어떤 순서로 연결되어야 하는지에 대한
문제점과 요청사항을 정리한 문서입니다.

## 2. 확인된 문제점

### 2-1. 어드민 대시보드 UI/UX 확인용 스크린샷 필요

- 현재 partner 화면의 실제 기준이 backend template인지, front 별도 구현인지 확인이 필요합니다.
- 특히 우측 `보안 가이드`, 우하단 챗봇, 로그인 후 대시보드 본문 구성은
  화면 기준이 조금만 달라도 바로 체감 차이가 발생합니다.

요청사항:

- [ ] partner dashboard의 최신 UI/UX 기준 스크린샷 또는 Figma 공유
- [ ] 현재 화면이 backend template 기준인지, front 별도 구현 기준인지 확인

`현재 공개 웹 페이지에서는 현재 파트너 대시보드 접근이 불가능해, front 측 자료 제공을 요합니다.`

### 2-2. 로그인 이후 선택 흐름 및 `분석 시작하기` 위치 재정의

- 현재 로그인 이후에는 단일 흐름으로 바로 진입하기보다,
  아래 3개 선택지가 있는 구조가 더 자연스럽습니다.
  - 기존 추천 스타일 확인
  - 트렌드 확인
  - 분석 시작하기
- 그런데 현재 `분석 시작하기`가 별도 CTA로 즉시 실행되면,
  위 선택지를 거치지 않고 바로 `촬영 -> 설문 -> 추천` 단계로 진입하게 됩니다.
- 이 경우 로그인 이후 화면에서
  - 선택형 진입
  - 즉시 실행형 진입
  두 방식이 동시에 존재하게 되어 사용자 흐름이 일관되지 않을 수 있습니다.

요청사항:

- [ ] 로그인 이후 기본 진입을 `3개 선택지 화면`으로 둘지 결정
- [ ] `분석 시작하기`를 선택지 중 하나로만 둘지 결정
- [ ] 별도 즉시 실행 CTA를 유지할지 결정
- [ ] 즉시 실행 CTA를 유지한다면 선택지 화면과의 우선순위 및 노출 위치를 정리
- [ ] 선택형 구조를 채택한다면 이를 초기 진입 화면으로 사용할지 결정

`그러니까 현재 문제는, 기존에는 선택지 3개가 출력되어야 했는데 지금은 로그인 직후 사진 촬영으로 넘어간다는 점입니다. 이 부분에 대해서 front와 back 모두 현재 작업 상태를 점검할 필요가 있을 것 같습니다.`

### 2-3. client 로그인 이후 partner 로그인 화면이 다시 출력되는 오류

- 현재 확인된 흐름 중, client 로그인 이후 설문 또는 분석 진입 과정에서
  partner 로그인 화면이 다시 출력되는 케이스가 있었습니다.
- 이 문제는 선택지 구조와는 별개로,
  실제 라우팅 또는 세션 전환 과정에서 잘못된 페이지가 노출되는 오류로 봐야 합니다.
- 특히 의도한 흐름이 `client 로그인 -> 설문/분석 진입`인데,
  중간에 `partner 로그인` 화면이 다시 뜨면 사용자는 현재 자신이 어떤 흐름에 있는지 알기 어렵습니다.

요청사항:

- [ ] 해당 재현 경로를 front 기준으로 다시 확인
- [ ] 오류가 발생하는 정확한 진입 순서와 최종 URL 공유
- [ ] client 흐름 중 partner 로그인 화면이 출력되면 안 되는 것이 맞는지 기준 확정
- [ ] backend 라우팅 오류인지, front 화면 전환 오류인지 함께 재확인

## 3. 판단

현재 이슈들은 단순 구현 누락보다는,
`partner 화면에서 어떤 행동을 유도할지`와 `customer 분석 흐름을 어떤 방식으로 시작할지`에 대한
기준이 아직 완전히 맞지 않은 부분, 그리고 일부 실제 라우팅 오류가 함께 섞여 있는 것으로 보입니다.

즉, 지금 단계에서는

- partner 화면의 CTA 목적
- 로그인 후 어디로 이동해야 하는지
- 로그인 이후 무엇을 먼저 보여줘야 하는지
- client 흐름 중 partner 로그인 화면이 다시 나오지 않도록 기준을 분리하는 것

를 먼저 합의해야 이후 화면 수정이 흔들리지 않습니다.

## 4. 프론트에 요청드리는 내용

- [ ] partner dashboard의 최신 기준 화면을 공유
- [ ] 로그인 이후 기본 진입을 `3개 선택지 화면`으로 둘지 결정
- [ ] `분석 시작하기`를 선택지 중 하나로만 둘지, 별도 즉시 실행 CTA로 둘지 결정
- [ ] partner 페이지에서 `분석 시작하기` 클릭 시 실제로 어떤 흐름을 시작할지 결정
- [ ] client 흐름에서 partner 로그인 화면 재출력이 발생하는 경로를 함께 확인

## 5. 비고

현재 백엔드에서는 partner / customer 흐름과 일부 화면 요소를 이미 연결해두었지만,
프론트에서 실제 확인한 화면과 다르게 느껴지는 부분은
별도 UI 기준을 다시 맞춰야 해결될 가능성이 큽니다.

따라서 본 리포트는 “어느 쪽이 틀렸는지”를 단정하기보다,
프론트 화면 기준을 먼저 확정해 다음 수정 범위를 줄이기 위한 요청서로 봐주시면 됩니다.
