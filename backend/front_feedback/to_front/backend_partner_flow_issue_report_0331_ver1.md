# 파트너 센터 흐름 문제점 및 요청사항 리포트

작성일: 0331  
버전: ver1

## 1. 기준 환경

현재 이 리포트의 검토 기준은 **Supabase 원격 DB + 최신 인증 구조**입니다.

즉, 예전의 `PIN 단일 로그인`이 아니라 아래 흐름을 기준으로 봅니다.

- shop 로그인: `사업자등록번호 + 비밀번호`
- 디자이너 로그인: `shop 로그인 -> 디자이너 선택 -> PIN`

## 2. 확인된 문제점

### 2-1. partner dashboard 최신 UI 기준 자료 필요

- 현재 partner 화면의 실제 기준이 backend template인지, front 별도 구현인지 다시 확인이 필요합니다.
- 특히 owner 화면과 designer 화면은 역할이 다르므로, 최종 UI 기준을 분리해서 보는 편이 안전합니다.

요청사항:

- [ ] partner dashboard 최신 스크린샷 또는 Figma 공유
- [ ] owner 화면과 designer 화면을 각각 어떤 구조로 볼지 확인

### 2-2. 로그인 이후 선택 구조 재정의 필요

- 현재 로그인 이후에는 아래 선택 구조가 더 자연스럽습니다.
  - 기존 추천 스타일 확인
  - 트렌드 확인
  - 분석 시작하기
- 그런데 `분석 시작하기`가 별도 CTA로 즉시 실행되면, 위 선택지를 거치지 않고 바로 `촬영 -> 설문 -> 추천`으로 진입하게 됩니다.
- 이 경우 선택형 진입과 즉시 실행형 진입이 섞여 사용자 흐름이 일관되지 않을 수 있습니다.

요청사항:

- [ ] 로그인 이후 기본 진입을 `선택형 화면`으로 둘지 결정
- [ ] `분석 시작하기`를 선택지 중 하나로만 둘지 결정
- [ ] 별도 즉시 실행 CTA를 유지할지 결정
- [ ] 즉시 실행 CTA를 유지한다면 우선순위와 노출 위치를 정리

### 2-3. client 흐름 중 partner 로그인 화면 재출력 여부 재확인

- client 흐름에서 partner 로그인 화면이 다시 보인다는 제보는 실제 UX 기준에서 큰 혼선을 만듭니다.
- 현재 backend 기준 고객 흐름은 `정보 입력 -> 사진 촬영 -> 설문 -> 결과`이므로, 중간에 partner 로그인 화면이 다시 뜨는 것은 의도된 동작으로 보기 어렵습니다.

요청사항:

- [ ] 재현 경로와 최종 URL을 다시 확인
- [ ] front 화면 전환 문제인지, backend 라우팅 문제인지 구분
- [ ] 이 현상이 최신 build에서도 재현되는지 확인

### 2-4. 사진 촬영 이전 선택지 부재

- 현재 고객 흐름에는 사진 촬영 전에 `기존 추천`, `트렌드`, `분석 시작하기`를 선택하는 별도 화면이 없습니다.
- 이 부분은 단순 백엔드 누락이라기보다, front와 back이 함께 확정해야 할 UX 구조에 가깝습니다.

요청사항:

- [ ] 사진 촬영 전 선택형 진입 화면이 필요한지 결정
- [ ] 트렌드 확인을 고객 사전 선택지에 넣을지 결정
- [ ] 기존 추천 확인을 고객 사전 선택지로 둘지 결정

## 3. 판단

현재 이슈들은 단순 버그 하나보다는,

- partner 화면에서 무엇을 하게 할지
- 로그인 이후 무엇을 먼저 보여줄지
- 고객 분석 흐름을 선택형으로 둘지 즉시 실행형으로 둘지

기준이 완전히 정리되지 않아 발생하는 부분이 큽니다.

## 4. 프론트에 요청드리는 내용

- [ ] owner/designer 기준 화면 자료 공유
- [ ] 로그인 이후 선택 구조 확정
- [ ] `분석 시작하기` CTA 위치와 역할 확정
- [ ] client 흐름 중 partner 로그인 재출력 현상 재확인

## 5. 비고

현재 backend는 인증 구조와 권한 분리, 고객 흐름 순서까지 최신 기준으로 반영한 상태입니다.
남은 핵심은 프론트 화면 기준을 맞추고, 사용자 여정을 어디서 분기시킬지 확정하는 일입니다.
