## 1. 기준 환경

현재 수동 검증 기준 환경은 **Supabase 원격 DB**입니다.

- 테스트 계정은 Supabase DB에 시드 적용을 마쳤습니다.
- 로컬 DB는 재현용 보조 환경으로만 사용합니다.
- 따라서 프론트 확인도 가능하면 Supabase 기준 화면과 흐름을 먼저 맞춰 보는 편이 정확합니다.

## 2. 현재 백엔드 기준 반영 상태

- shop 로그인: `사업자등록번호 + 비밀번호`
- 디자이너 로그인: `shop 로그인 -> 디자이너 선택 -> PIN`
- shop owner 대시보드: `/partner/dashboard/`
- designer 전용 화면: `/partner/staff/`
- 고객 흐름: `/customer/ -> /customer/camera/ -> /customer/survey/ -> /customer/result/`
- 회원가입 성공 후 `/partner/dashboard/` 즉시 이동
- `errors`, `message`, `error_code` 응답 형식 제공

## 3. 프론트에서 우선 확인이 필요한 항목

### 3-1. 파트너 로그인/권한 구조

- [ ] shop 로그인 화면이 `사업자등록번호 + 비밀번호` 구조로 맞는지 확인
- [ ] shop 로그인 후 디자이너 선택 화면이 보이는지 확인
- [ ] shop owner가 매장 전체 대시보드로 진입 가능한지 확인
- [ ] designer는 `/partner/staff/` 기준 화면으로 진입하는지 확인
- [ ] designer 화면에서 owner 전용 요소가 노출되지 않는지 확인

### 3-2. 회원가입 화면

- [ ] `agree_third_party_sharing` 체크박스와 문구가 실제 화면에 있는지 확인
- [ ] 사업자등록번호 오류가 필드 단위로 보이는지 확인
- [ ] 중복 전화번호 오류가 필드 단위로 보이는지 확인
- [ ] 상단 요약 문구가 일반 문구로만 고정되지 않는지 확인
- [ ] 회원가입 성공 후 `/partner/dashboard/`로 즉시 이동하는지 확인

### 3-3. 고객 흐름

- [ ] 현재 고객 흐름이 `정보 입력 -> 사진 촬영 -> 설문 -> 결과` 순서로 보이는지 확인
- [ ] 촬영 후 설문으로 자연스럽게 넘어가는지 확인
- [ ] 결과 페이지에서 mock/local 결과와 실제 결과 구분이 필요한지 확인

## 4. 프론트와 함께 정리해야 하는 이슈

### 4-1. 파트너 화면 기준

- partner/dashboard 최신 UI 기준안 공유 필요
- owner와 designer 화면 차이를 어떤 수준으로 둘지 확인 필요
- 보안 가이드, 챗봇, 탭 구성 등 시각 요소 기준 재확인 필요

### 4-2. 로그인 이후 선택 구조

- 로그인 직후 곧바로 분석으로 갈지
- `기존 추천`, `트렌드`, `분석 시작하기` 선택형 화면을 둘지
- `분석 시작하기`를 별도 CTA로 유지할지

이 부분은 UI/UX 결정이 선행되어야 backend도 맞춰서 정리할 수 있습니다.
