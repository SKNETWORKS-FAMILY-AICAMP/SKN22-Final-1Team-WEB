1. 컴포넌트 다이어그램
1) 구성 요소
클라이언트(Client):
참여자: 미용사, 손님
환경: PC, 태블릿, 키오스크, 모바일 등 디바이스별 반응형 UI
기술: Vanilla JS, HTML, CSS
AWS Cloud:
참여자: 미용사, 손님
컴포넌트: S3, EC2, ECS
역할: 이미지 저장 및 웹/AI 모델 서버 운영
Backend (웹서버 백엔드):
컴포넌트: Python, FastAPI
역할: 설문 수신 및 벡터화, 얼굴 분석 요청, 추천 결과 조합, API 제공
AI Server (AI 서버):
컴포넌트: PyTorch, MediaPipe, HairCLIPv2, SAM2
역할: 얼굴 랜드마크 추출, 비율 계산, 얼굴 영역 마스킹, 가상 헤어스타일 합성 이미지 생성
데이터베이스 (DB 및 Vector DB):
DB: PostgreSQL (고객 프로파일, 설문 응답, 시술 이력 저장)
Vector DB: chroma (user_preference_vector 및 헤어스타일 임베딩 저장)
2) 설명
클라이언트 프론트엔드에서 미용사나 손님이 입력한 취향 정보, 개인 정보, 사진 데이터를 기반으로 작업을 시작합니다.
웹서버 백엔드(FastAPI)는 전달받은 설문 응답을 벡터화하고 고객 데이터를 PostgreSQL DB에 저장합니다.
AI 서버는 수신한 사진과 취향 벡터를 바탕으로 MediaPipe를 활용해 얼굴형을 분석하고, Vector DB(chroma)를 통해 취향과 얼굴형에 부합하는 Top-5 스타일을 매칭합니다.
매칭된 헤어스타일은 HairCLIPv2 모델을 활용하여 고객 원본 사진의 얼굴 영역에 맞춰 가상 합성 처리되며, 최종 생성된 시뮬레이션 이미지를 클라이언트로 반환합니다.

2. 시퀀스 다이어그램
1) 참여자 (Actors)
User: 미용사 또는 손님 (기본 정보 입력, 취향 설문 응답, 정면 사진 촬영)
웹서버 백엔드 (FastAPI): 비즈니스 로직 처리, API 제공, 데이터 연동 요청 관리
AWS S3: 촬영된 고객 얼굴 사진을 임시 보관하는 스토리지
Vector DB (chroma): 설문 기반 취향 벡터 및 헤어스타일 임베딩 저장, 코사인 유사도 검색 연산.
AI 서버 (MediaPipe, HairCLIPv2, SAM2): 얼굴 특징 및 비율 분석, 정밀 마스킹, 가상 시뮬레이션 이미지 생성 기능 수행
PostgreSQL DB: 고객별 정보, 개인 취향 설문, 매칭 이력 및 시술 데이터의 영구 저장
2) 주요 흐름
클라이언트(프론트엔드)에서 사용자가 상담 시작 후 고객 정보와 5가지 항목(길이, 분위기, 모발 상태, 컬러, 예산)의 취향 설문을 입력합니다.
클라이언트 카메라를 활성화하여 고객의 정면 얼굴 사진을 촬영하고 해당 사진을 AWS S3에 임시 업로드합니다.
웹서버 백엔드가 취향 설문 응답 데이터를 원-핫 인코딩하여 user_preference_vector를 생성합니다.
웹서버가 AI 서버로 해당 벡터 데이터와 얼굴 분석 및 추천을 요청합니다.
AI 서버의 MediaPipe(FaceMesh) 모델이 468개 랜드마크 좌표를 추출하여 이마·광대·턱 너비 등을 포함한 얼굴 비율 벡터 및 황금비율 근접도를 계산합니다.
추출된 얼굴 비율 벡터(40%), 황금비율 근접도(20%), 취향 벡터(40%)를 가중 결합하여 FAISS 내의 헤어스타일 임베딩과 코사인 유사도를 비교해 최적의 스타일 Top-5를 추출합니다.
AI 서버는 HairCLIPv2 모델을 사용해 추천된 스타일을 고객 얼굴 이미지에 합성합니다.
생성된 Top-5 스타일별 시뮬레이션 결과 이미지가 백엔드를 거쳐 프론트엔드 화면에 표시됩니다.
고객이 시뮬레이션을 통해 최종 스타일을 확정하면 PostgreSQL DB에 고객 및 시술 기록으로 저장됩니다.

3. 액티비티 다이어그램
1) 구성
시작 지점: 키오스크나 태블릿에서 상담 시작 버튼을 클릭하거나, 기존 고객 여부를 확인하며 서비스에 진입 
클라이언트 활동(입력): 이름 및 연락처 입력, 5개 항목 취향 설문 진행, 얼굴 사진 촬영 및 재촬영 확인
데이터 전송 및 변환 활동: 수집된 입력 정보 및 사진을 기반으로 백엔드 서버에서 취향 정보를 벡터화
AI 얼굴 분석 활동: 사진 데이터를 MediaPipe로 분석하여 5가지(타원형, 둥근형, 각형, 역삼각형, 긴형) 얼굴형 분류 수행 
추천 연산 활동: 얼굴형 분류 결과와 취향 벡터를 결합하여 추천 엔진에서 내부 정수표 및 DB 유사도를 계산
이미지 합성 활동: Top-5 스타일에 대한 이미지 생성 요청 수행 (HairCLIPv2 가상 헤어 합성)
결과 출력 활동: 헤어스타일 리스트와 추천 근거, 그리고 시뮬레이션 결과 이미지를 사용자에게 반환
DB 저장 활동: 확정된 스타일의 이력을 고객 정보와 연결하여 PostgreSQL 및 Vector DB에 기록 (개인정보 암호화 및 비식별화 처리 포함)
종료 지점: 시술용 참고 이미지 제공 및 상담 종료
2) 참여자 (Actors)
클라이언트 활동(입력 노드): 미용사 및 손님을 통한 취향 정보, 개인 정보, 사진 입력 작업
웹서버/AI서버 활동(처리 노드): 취향 정보 벡터 생성, 얼굴형 분석, 얼굴형에 맞는 헤어스타일 리스트 도출 및 내부 정수표 대조, 이미지 생성(가상 합성) 요청
클라이언트 활동(출력 노드): 최종 헤어스타일 리스트 및 시뮬레이션 화면 출력 제공
