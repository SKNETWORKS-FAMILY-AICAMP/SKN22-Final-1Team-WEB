# [Report] Frontend 연동 준비 상태 재검토 보고서

작성일: 2026-03-30  
재검토 반영: 0331

## 1. 목적

프론트와 백엔드의 현재 연동 준비 상태를 최신 기준으로 다시 정리한 문서입니다.

이번 재검토 기준은 다음과 같습니다.

- backend 인증 구조 최신화 반영
- Supabase 원격 DB 기준 검증
- session 기반 partner/customer 흐름 기준 확인

## 2. 최신 기준에서 바뀐 점

- partner 로그인은 더 이상 `PIN 단일 입력` 구조가 아닙니다.
- shop 로그인 후 디자이너 선택 단계가 추가되었습니다.
- owner와 designer는 같은 화면이 아니라 역할이 분리된 화면으로 봐야 합니다.
- 고객 흐름은 `정보 입력 -> 촬영 -> 설문 -> 결과` 순서입니다.

## 3. 프론트 연동 체크 포인트

- shop 로그인 화면이 현재 인증 구조와 맞는지
- 디자이너 선택 화면이 실제로 필요한지
- owner와 designer UI를 분리해서 표현할지
- 회원가입 오류 메시지를 `errors` 기준으로 보여주는지
- 결과 화면이 mock/local 흐름이 아니라 최신 backend 흐름과 맞는지

## 4. 현재 판단

백엔드 쪽 구조 변경은 큰 틀에서 반영이 끝난 상태이고,
이제 프론트는 화면 구조와 사용자 흐름을 최신 인증 구조에 맞춰 확인하는 단계에 가깝습니다.

즉, 지금 프론트 확인의 핵심은 `토큰 저장 전략`보다도
현재 session 기반 화면 전환과 역할 분리가 실제 UX에서 자연스러운지 점검하는 것입니다.
