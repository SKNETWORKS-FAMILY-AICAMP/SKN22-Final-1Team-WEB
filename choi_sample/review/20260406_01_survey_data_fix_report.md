# [보고서] 디자이너 대시보드 고객 리스트 및 취향 분석 데이터 이슈 분석 및 조치

**작성일:** 2026-04-06
**대상 고객:** 고객 ID 4987 (理)
**상태:** 조치 완료 (재발 방지 적용)

---

## 1. 개요

디자이너 대시보드(Partner Portal)에서 특정 고객(4987)의 리스트에 '세션 없음' 메시지가 표시되고, 상세 페이지에서 '스타일 취향 분석' 데이터가 정상적으로 노출되지 않는 문제에 대한 원인 분석 및 해결 보고입니다.

## 2. 현상 및 원인 분석

### 2.1. '세션 없음' 메시지 표시

- **현상:** 고객 이름 아래 '세션 없음' 배지 표시
- **원인:** 해당 고객과 연결된 **'활성 상담(Active Consultation)'** 데이터가 데이터베이스에 존재하지 않기 때문입니다.
- **분석 결과:** 상담이 정상적으로 종료되었거나, 고객이 키오스크에서 '상담 신청' 단계까지 완료하지 않았을 때 발생합니다. 세션이 없으면 '진단 카드' 작성이 제한됩니다.

### 2.2. '스타일 취향 분석' 데이터 누락 (ID 4987)

- **현상:** 설문 결과가 모두 `'unknown'`으로 표시되며, 분석 데이터가 비어 있음
- **근본 원인 (Technical Bug):**
  - 데이터베이스의 성별 코드는 `'M'`, `'F'`로 저장되어 있었으나, 설문 템플릿(`survey.html`)은 `'male'`, `'female'` 문자열로만 성별을 판별하고 있었습니다.
  - 이에 따라 남성 고객(4987)에게 **여성용 설문지**가 잘못 노출되었습니다.
  - 제출된 여성용 답변 데이터가 백엔드 서버의 남성용 매핑 로직과 충돌하여, 모든 값이 `'unknown'`으로 처리되어 저장되었습니다.

## 3. 조치 내용

### 3.1. 성별 정보 정규화 (Hotfix)

- **파일:** `app/front_views.py`
- **수정사항:** 설문 페이지 진입 시 성별 정보를 문자열 포맷과 무관하게 `'male'`, `'female'`로 정규화하여 템플릿에 전달하도록 로직을 강화했습니다.
- **결과:** 이제 모든 성별 포맷(`M`, `male`, `남성` 등)에서 고객에게 맞는 설문지가 정상적으로 노출됩니다.

### 3.2. 데이터 복구 관련

- 이미 `'unknown'`으로 저장된 4987번 고객의 기존 데이터는 원본 답변이 보관되지 않아 복구가 불가능합니다.
- **해결책:** 해당 고객이 대시보드의 **'재분석'** 기능을 통해 설문을 다시 진행하면, 수정된 로직에 의해 데이터가 정상적으로 저장되고 노출됩니다.

## 4. 향후 대응 및 권고

- **재분석 안내:** 취향 데이터가 비어 있는 고객의 경우, 디자이너 대시보드 상세 페이지 우측 상단의 '재분석' 버튼을 눌러 설문을 다시 진행하도록 안내 바랍니다.
- **로그인 프로세스 점검:** 향후 성별 코드와 같은 기본 정보의 일관성을 위해 백엔드-프론트엔드 간의 데이터 규격을 재확인할 예정입니다.
